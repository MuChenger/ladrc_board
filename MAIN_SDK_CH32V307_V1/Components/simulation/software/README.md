# 控制算法模拟器-by嵌入式新起点 使用说明

## 1. 工具简介

本目录提供一个基于 PyQt5 的控制算法模拟器-by嵌入式新起点，用于“下位机控制算法 + 上位机被控对象仿真”的联调测试。

它的核心作用是：

- 通过串口或蓝牙串口透传与下位机通信
- 接收下位机上报的姿态、控制量、运行状态等遥测数据
- 在上位机内部运行一套单自由度深度/高度对象仿真
- 将仿真得到的反馈量再发送给下位机，形成闭环
- 通过图形界面完成算法切换、参考值设置、运行控制、状态观察与 CSV 记录

当前入口程序为 [run.py](g:\ladrc_board\MAIN_SDK_CH32V307_V1\Components\simulation\software\run.py)。

## 6. 界面说明

### 6.1 串口区域

用于与下位机建立通信连接。

- 端口：选择目标串口号
- 波特率：选择串口波特率，默认 `115200`
- 刷新：重新扫描系统可用串口
- 连接：打开串口
- 断开：关闭串口
- 二进制发送：勾选后，上位机向下位机发送仿真反馈时使用二进制帧；取消后使用文本格式

状态栏会显示当前连接状态，例如：

- `未连接`
- `已连接: COM5@115200`

### 6.2 控制台区域

用于发送控制命令和查看日志输出。

- 算法：
  - `PID`
  - `LADRC`
  - `开环`
- 应用算法：发送算法切换命令
- 参考值：输入目标值
- 设置参考值：发送参考值命令
- 启动：发送 `RUN 1`
- 停止：发送 `RUN 0`
- 获取状态：发送 `GET STATUS`
- 手工命令输入框：可直接输入原始命令
- 下方文本框：显示发送记录、应答、普通日志和错误信息

说明：

- 下拉框显示为中文，但实际发送给下位机的仍是协议要求的英文命令值
- “开环”实际发送为 `OPEN_LOOP`

### 6.3 运行状态区域

用于显示当前运行信息。

主要字段包括：

- 算法
- 运行状态
- 参考值
- 反馈值
- 控制输出
- 接收帧数
- 发送帧数
- 解析错误
- 延迟
- 通信超时

其中：

- “运行状态”为 `运行中` 或 `空闲`
- “通信超时”会在长时间未收到遥测数据时显示为 `超时`

### 6.4 波形区域

用于显示实时曲线。

当前支持的通道包括：

- 参考值
- 反馈值
- 控制输出
- 滚转角
- 俯仰角
- 偏航角
- 深度变化率
- 扰动

支持以下操作：

- 勾选 / 取消勾选通道
- 调整显示时间窗口
- 暂停 / 继续刷新
- 清空当前曲线

### 6.5 录制区域

用于保存运行过程中的数据。

- 开始录制：选择 CSV 保存路径并开始记录
- 停止录制：结束记录并关闭文件

记录过程中，界面会显示当前文件路径。

### 6.6 3D 模型区域

用于显示简化的三维姿态模型。

- 支持三种显示模式：
  - 姿态模式
  - 轨迹模式
  - 水下场景模式
- 支持三种模型库：
  - 水下机器人
  - 飞行器
  - 通用载体
- 每种模型包含独有动态部件动画：
  - 水下机器人推进器
  - 飞行器机头螺旋桨
  - 通用载体旋转环
- 水下场景模式额外包含环境特效：
  - 海床起伏地形
  - 上浮气泡
  - 前灯呼吸闪烁
- 支持外部模型加载
  - 当前支持 `OBJ`、`STL`、`OFF` 三种格式
  - 点击 3D 面板中的“导入模型”可选择本地模型文件
  - 未导入外部模型时，系统默认使用内置 ROV 模型
  - 点击“默认模型”可随时切回系统内置模型
  - 导入后可在界面中直接调整模型滚转、俯仰、偏航和缩放
  - 可调整导入模型的颜色、透明度和边线显示
  - 可通过“重置模型姿态”恢复默认导入朝向
  - 可通过“重置外观”恢复默认显示参数
  - 建议将常用模型放在 `assets/models/rov/` 目录，便于集中管理
- 根据下位机遥测中的 `roll`、`pitch`、`yaw` 实时旋转模型
- 根据当前仿真深度更新深度读数
- 显示滚转、俯仰、偏航、深度数值
- 与波形区同时显示，便于一边看曲线一边看姿态
- 支持轨迹尾迹显示
- 支持视角跟随、重置视角和清空轨迹

## 7. 推荐使用流程

## 7.1 仅启动界面

1. 安装依赖
2. 执行 `python run.py`
3. 确认界面正常打开

## 7.2 与下位机联调

1. 打开软件
2. 点击“刷新”，选择正确串口
3. 设置与下位机一致的波特率
4. 点击“连接”
5. 观察状态是否变为“已连接”
6. 选择算法并点击“应用算法”
7. 设置参考值
8. 点击“启动”
9. 观察右侧波形、运行状态和控制台输出

## 7.3 数据记录

1. 在运行前或运行中点击“开始录制”
2. 选择 CSV 输出路径
3. 完成测试后点击“停止录制”
4. 使用 Excel、Python 或其他工具分析导出的 CSV

## 8. 支持的命令

控制台支持直接输入原始命令。当前界面内置了以下常用操作：

- `ALG PID`
- `ALG LADRC`
- `ALG OPEN_LOOP`
- `SET REF <val>`
- `RUN 1`
- `RUN 0`
- `GET STATUS`

你也可以在输入框手工发送例如：

```text
SET KP 1.2
SET KI 0.05
SET KD 0.1
SAVE
```

实际是否支持，由下位机固件命令解析逻辑决定。

## 9. 通信数据格式

本程序同时支持二进制帧和文本行两种输入格式。

### 9.1 下位机 -> 上位机

可接收以下两类数据：

1. 二进制遥测帧
2. 文本遥测 / 文本应答

#### 二进制帧格式

帧头定义：

```text
AA 55 | msg_id | len | seq | payload | crc16
```

其中：

- `msg_id = 0x01` 表示遥测数据
- `msg_id = 0x20` 表示应答文本

遥测载荷当前按以下结构解析：

```text
<IfffffBB
```

字段顺序为：

- `timestamp_ms`
- `roll`
- `pitch`
- `yaw`
- `u_cmd`
- `ref`
- `algo_id`
- `run_state`

说明：

- 当前二进制遥测中未单独包含 `feedback`
- 如果下位机使用文本格式上报，则可以带上 `feedback`

#### 文本行格式

支持以下形式：

```text
timestamp=123456,roll=0.1,pitch=0.2,yaw=0.3,u_cmd=1.0,ref=2.0,feedback=1.8,algo_id=1,run_state=1
```

或：

```json
{"timestamp":123456,"roll":0.1,"pitch":0.2,"yaw":0.3,"u_cmd":1.0,"ref":2.0,"feedback":1.8,"algo_id":1,"run_state":1}
```

普通应答也支持直接发送：

```text
OK
ERR <code> <msg>
```

推荐遥测字段：

- `timestamp`
- `roll`
- `pitch`
- `yaw`
- `u_cmd`
- `ref`
- `feedback`
- `algo_id`
- `run_state`

### 9.2 上位机 -> 下位机

上位机会周期性发送仿真反馈数据，内容包括：

- `timestamp`
- `depth`
- `depth_rate`
- `disturbance`

若使用二进制模式，发送的消息类型为：

- `msg_id = 0x10`

若使用文本模式，则发送格式类似：

```text
timestamp=123456,depth=0.12345,depth_rate=0.01234,disturbance=0.10000
```

## 10. 仿真模型说明

当前内置的是一个单自由度深度对象模型，主要特性如下：

- 包含质量项
- 包含阻尼项
- 包含浮力偏置项
- 包含周期扰动
- 包含测量噪声

上位机根据下位机输出的控制量 `u_cmd` 推进仿真，并将生成的：

- 深度 `depth`
- 深度变化率 `depth_rate`
- 扰动 `disturbance`

发送回下位机，构成简化闭环。

除波形外，界面还会以 3D 模型形式直观显示姿态变化。

如需获得更接近真实 BlueROV 的外观，建议提供或放置真实的 `OBJ`、`STL` 或 `OFF` 模型文件，而不是继续依赖内置程序化模型。

## 11. CSV 记录内容

CSV 记录会根据首次写入的数据字段自动建立表头。

通常会包含以下数据：

- `pc_time_ms`
- `timestamp_ms`
- `roll`
- `pitch`
- `yaw`
- `u_cmd`
- `ref`
- `feedback`
- `algo_id`
- `run_state`
- `sim_depth`
- `sim_depth_rate`
- `sim_disturbance`

说明：

- 遥测记录和仿真记录会写入同一个 CSV
- 首次写入时出现的字段会固定为该文件表头

## 12. 常见问题

### 12.1 程序无法启动

请先确认依赖是否已经安装：

```powershell
python -m pip install -r requirements_pyqt.txt
```

### 12.2 提示缺少 `pyqtgraph`

说明图形显示依赖未安装，重新执行依赖安装即可。

### 12.3 能打开界面但没有波形

请检查：

- 是否已经连接到正确串口
- 下位机是否确实在发送遥测
- 遥测字段名称是否符合程序支持的格式
- 文本数据是否以换行结尾

### 12.4 显示“通信超时”

说明程序在设定时间内未收到新的遥测数据。请检查：

- 串口连接是否中断
- 波特率是否匹配
- 下位机发送频率是否过低
- 下位机协议格式是否正确

### 12.5 点击“开环”后下位机不响应

请确认下位机是否支持命令：

```text
ALG OPEN_LOOP
```

界面显示为“开环”，但协议实际发送值仍为 `OPEN_LOOP`。
