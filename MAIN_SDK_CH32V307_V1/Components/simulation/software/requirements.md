# 控制算法仿真软件需求文档

## 1. 文档目的
本文档用于整理“下位机实机控制 + 上位机被控对象仿真”的需求，作为后续设计与开发依据。

## 2. 项目背景与目标
- 下位机为本地开发板（含 BMI160），可实时获取姿态并周期运行控制算法。
- 上位机用于仿真被控对象（示例为水下机器人），并与下位机进行双向数据交互。
- 通信链路为蓝牙串口透传，可等效为普通串口通信。
- 目标是构建一个可用于控制算法联调、切换、观测和验收的仿真软件系统。

## 3. 系统范围
- 在范围内：
- 下位机控制算法运行与执行。
- 上位机被控对象动力学仿真。
- 上下位机串口协议交互。
- 上位机命令行控制与波形可视化（VOFA+风格）。
- 数据记录、回放与性能评估。
- 不在范围内（当前阶段）：
- 实际水下机器人硬件执行机构联动。
- 云端远程平台接入。

## 4. 系统角色与分工
- 下位机（开发板）：
- 采集 BMI160 姿态数据。
- 周期运行控制算法（如 PID/LADRC）。
- 输出控制量 `u`。
- 接收上位机反馈状态与控制命令。
- 上位机（仿真软件）：
- 模拟被控对象（如深度/高度模型）。
- 提供字符命令交互界面。
- 提供多通道实时波形显示。
- 记录、导出和回放数据。

## 5. 核心功能需求
### 5.1 闭环仿真
- 系统应支持下位机控制算法与上位机仿真对象形成闭环。
- 上位机应向下位机反馈被控量（如深度/高度）和必要状态量。

### 5.2 算法切换与参数配置
- 上位机可选择下位机使用的控制算法。
- 用户可通过字符命令控制底层算法。
- 至少支持算法类型：
- `PID`
- `LADRC`
- `OPEN_LOOP`（开环/调试模式）
- 参数设置应包含合法性校验（范围、类型、单位）。

### 5.3 字符命令交互（CLI）
- 系统应支持命令下发、应答显示、错误提示、历史命令。
- 推荐命令集（首版）：
- `ALG PID` / `ALG LADRC` / `ALG OPEN_LOOP`
- `SET KP <val>`、`SET KI <val>`、`SET KD <val>`
- `SET REF <val>`
- `RUN 1`、`RUN 0`
- `GET STATUS`
- `SAVE`
- 每条命令必须有响应：`OK` 或 `ERR <code> <msg>`。

### 5.4 波形显示（VOFA+风格）
- 上位机应提供类似 VOFA+ 的多通道实时曲线显示能力。
- 支持功能：
- 通道开关、颜色配置、命名、分组。
- 时间窗口滚动（建议 10~30 s）。
- 缩放、游标、暂停、清屏。
- 事件标记（算法切换、目标值变更、运行状态切换）。
- 推荐必显通道：
- `ref`、`feedback(depth/height)`、`u`
- 推荐可选通道：
- `roll`、`pitch`、`yaw`、`error`、`disturbance`、`algo_id`

### 5.5 数据记录与回放
- 支持一键开始/停止记录。
- 支持导出 CSV（时间戳 + 各通道）。
- 支持离线回放并复现关键事件标记。

## 6. 通信与协议需求
### 6.1 传输方式
- 蓝牙串口透传，按 UART 链路处理。

### 6.2 协议建议
- 二进制帧建议字段：
- 帧头（如 `0xAA 0x55`）
- 消息类型 `msg_id`
- 长度 `len`
- 序号 `seq`
- 时间戳 `timestamp`
- 载荷 `payload`
- 校验 `crc16`

### 6.3 数据方向
- 下位机 -> 上位机（示例）：
- `timestamp, roll, pitch, yaw, u_cmd`
- 上位机 -> 下位机（示例）：
- `timestamp, depth/height, depth_rate, disturbance`

### 6.4 时序与频率
- 控制循环与通信应解耦。
- 推荐控制周期：100 Hz（下位机）。
- 推荐通信更新：20~50 Hz（双向）。

## 7. 状态机与安全约束
### 7.1 建议状态机
- `IDLE`：未运行控制。
- `RUN`：闭环运行。
- `FAULT`：通信超时或参数错误等故障状态。

### 7.2 安全策略
- 串口通信超时（如 >300 ms）应触发安全策略：
- 控制量清零或限幅保持（按最终设计确定）。
- 运行中算法切换应受保护：
- 建议仅允许在 `IDLE` 切换；或执行软切换并限斜率。
- 非法命令或参数不得破坏当前稳定运行状态。

## 8. 被控对象仿真需求（上位机）
- 首版可采用单自由度深度/高度模型。
- 应支持：
- 参数配置（质量、阻尼、浮力项等）。
- 扰动注入（水流、随机噪声）。
- 传感器噪声与滞后模拟（可选）。

## 9. 非功能需求
- 实时性：50 Hz 波形刷新下无明显卡顿。
- 稳定性：连续运行 10 分钟不崩溃。
- 可观测性：应显示采样率、丢包率、通信延迟。
- 可维护性：命令与通道定义可扩展。

## 10. 验收标准（首版）
- 闭环连通：
- 上下位机双向通信稳定，10 分钟无协议失步。
- 算法控制：
- 算法切换命令在 200 ms 内得到明确应答。
- `GET STATUS` 与实际运行状态一致。
- 波形能力：
- 至少支持 8 路通道同时显示。
- 曲线可连续滚动，支持暂停/缩放/游标。
- 容错能力：
- 丢包/超时条件下系统进入预期安全状态，不发散。
- 记录能力：
- CSV 记录完整且可用于离线回放。

## 11. 后续设计输入
- 串口协议字段与编码细节（最终定稿）。
- 命令字与参数字典（名称、单位、范围、默认值）。
- 被控对象数学模型与离散化方法。
- 上位机 UI 页面与交互流程图。
